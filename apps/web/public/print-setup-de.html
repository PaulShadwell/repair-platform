POS Etikett Druck Setup (macOS / Windows / Linux)

Folge diesen Schritten auf jedem POS Computer, damit Etiketten ueber die Cloud API auf dem lokalen Drucker gedruckt werden.

Bevor du startest

  • Der POS Drucker ist verbunden und im Betriebssystem als Drucker sichtbar.
  • Ein Testdruck aus den Druckereinstellungen funktioniert.
  • Node.js ist auf dem POS Computer installiert.

Node.js installieren (erforderlich)

Installiere Node.js LTS (empfohlen: Version 20 oder neuer), bevor du den Print Agent koppelst.

  • macOS: Download von nodejs.org und Installer ausfuehren.
  • Windows: Windows Installer von nodejs.org herunterladen und installieren.
  • Linux: Installation ueber Paketmanager oder NodeSource (LTS empfohlen). 
# Installation pruefen
node -v
npm -v

Print Agent Paket herunterladen

Lade das aktuelle OS Paket aus den GitHub Releases herunter: repair-platform releases

  • repair-print-agent-macos.zip
  • repair-print-agent-windows.zip
  • repair-print-agent-linux.zip

ZIP entpacken und im entpackten Ordner ein Terminal oeffnen.

1) Druckerprofil in Repair Platform waehlen

  1. In der App Einstellungen oeffnen.
  2. Das passende Druckerprofil fuer diesen POS waehlen.
  3. Kopplungscode erzeugen klicken.
  4. Code offen lassen (10 Minuten gueltig).

2) Lokalen Print Agent koppeln (einmalig)

macOS / Linux:

# im entpackten Print Agent Ordner
node agent.mjs pair \
  --api "https://repair-platform.techvee.cloud" \
  --code "<PAIRING_CODE>" \
  --name "POS-Laptop-1" \
  --printer "Label_TD80"

Windows (PowerShell):

# im entpackten Print Agent Ordner
node agent.mjs pair `
  --api "https://repair-platform.techvee.cloud" `
  --code "<PAIRING_CODE>" `
  --name "POS-Laptop-1" `
  --printer "Label_TD80"

Ersetze <PAIRING_CODE>, POS-Laptop-1 und Label_TD80 mit euren Werten.

Hinweis fuer Windows: als --printer einen freigegebenen Druckernamen oder \\host\printer angeben.

3) Print Agent starten

macOS / Linux:

node agent.mjs run

Windows (PowerShell):

.\run-agent.ps1

Der Agent holt Druckjobs aus der Cloud und druckt lokal am POS Computer.

4) Automatisch beim Login starten (empfohlen)

macOS (launchd):

chmod +x install-launchd.sh uninstall-launchd.sh
./install-launchd.sh

Windows (Task Scheduler):

.\install-startup-task.ps1

Linux (systemd user service):

chmod +x install-systemd-user.sh uninstall-systemd-user.sh
./install-systemd-user.sh

Pruefen / Fehlerbehebung

  • In den App Einstellungen sollte der Druckerstatus Online zeigen.
  • Bei Offline: node agent.mjs run erneut starten und Druckername pruefen.
  • Sicherstellen, dass in der App das passende Druckerprofil gewaehlt ist.
  • Unter Windows/Linux zuerst lokalen Druckerbefehl bzw. Treiber pruefen.
