POS Label Print Setup (macOS / Windows / Linux)

Follow these steps on each POS computer to print labels through the cloud API using the local printer.

Before you start

  • The POS printer is connected and visible in your operating system as a printer.
  • You can print a test page from the OS printer settings.
  • Node.js is installed on the POS computer.

Install Node.js (required)

Install Node.js LTS (version 20 or newer recommended) before pairing the print agent.

  • macOS: Download from nodejs.org and run the installer package.
  • Windows: Download the Windows installer from nodejs.org and install for all users.
  • Linux: Install from your package manager or NodeSource (LTS recommended). 
# verify installation
node -v
npm -v

Download the print agent package

Download the latest OS package from GitHub Releases: repair-platform releases

  • repair-print-agent-macos.zip
  • repair-print-agent-windows.zip
  • repair-print-agent-linux.zip

Extract the ZIP and open a terminal in the extracted folder.

1) Select the printer profile in Repair Platform

  1. Open Settings in the app.
  2. Select the correct Printer Profile for this POS device.
  3. Click Generate pairing code.
  4. Keep the code open (valid for 10 minutes).

2) Pair the local print agent (one-time)

macOS / Linux:

# inside the extracted print-agent folder
node agent.mjs pair \
  --api "https://repair-platform.techvee.cloud" \
  --code "<PAIRING_CODE>" \
  --name "POS-Laptop-1" \
  --printer "Label_TD80"

Windows (PowerShell):

# inside the extracted print-agent folder
node agent.mjs pair `
  --api "https://repair-platform.techvee.cloud" `
  --code "<PAIRING_CODE>" `
  --name "POS-Laptop-1" `
  --printer "Label_TD80"

Replace <PAIRING_CODE>, POS-Laptop-1, and Label_TD80 with your values.

Windows note: use a shared printer name (for example Label_TD80) or a full share path \\host\printer.

3) Run the print agent

macOS / Linux:

node agent.mjs run

Windows (PowerShell):

.\run-agent.ps1

The agent polls the cloud API and prints locally through the POS computer.

4) Auto-start at login (recommended)

macOS (launchd):

chmod +x install-launchd.sh uninstall-launchd.sh
./install-launchd.sh

Windows (Task Scheduler):

.\install-startup-task.ps1

Linux (systemd user service):

chmod +x install-systemd-user.sh uninstall-systemd-user.sh
./install-systemd-user.sh

Check / troubleshooting

  • In app Settings, printer status should show Online.
  • If it shows Offline, run node agent.mjs run again and check printer name.
  • Make sure the selected app profile matches the paired POS computer profile.
  • On Windows/Linux, make sure your local print command/driver is working from the OS first.
